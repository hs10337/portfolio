# AGENTS.md

## Project Overview

This is Belle's personal portfolio site.

The portfolio should position Belle as a product designer with strong systems thinking, AI product experience, visual craft, and a thoughtful editorial point of view. The site should feel chic, clean, modern, and quietly expressive — not overly corporate, not overly playful, and not generic SaaS.

Primary goals:

- Showcase high-quality product design work.
- Position Belle for AI SaaS, browser, design systems, and product strategy opportunities.
- Make room for personal and concept projects such as Bloom and Momento.
- Communicate range: product designer, design systems thinker, AI UX strategist, visual storyteller, and builder.
- Feel editorial, polished, soft-futuristic, and memorable.

## Brand Direction

The visual direction is **soft future editorial**.

The site should combine:

- Editorial typography
- Spacious layouts
- Subtle motion
- Soft gradients
- Refined pastel tones
- Occasional warm orange accents
- Clean product storytelling
- High visual confidence without feeling flashy

Avoid:

- Generic startup landing page aesthetics
- Overly playful cartoon styling
- Heavy glassmorphism
- Excessive animation
- Dark, cyberpunk AI visuals
- Dense case study layouts
- Template-like portfolio patterns

The portfolio should feel like a designer with taste, range, and strategic depth.

## Design Principles

Use these principles when making design or implementation decisions:

### 1. Editorial before dashboard

Layouts should feel composed and intentional, not like a SaaS admin panel unless the project itself requires it.

### 2. Soft, not weak

Use soft colors and gradients, but keep typography, spacing, and hierarchy sharp.

### 3. Motion should clarify identity

Animations should express range, transition, layering, or transformation. Avoid decorative motion that slows down comprehension.

### 4. Case studies should feel skimmable

Recruiters and design leaders should understand the project quickly. Use clear sections, strong captions, and visual rhythm.

### 5. Show range without feeling scattered

Personal projects and professional AI work should feel connected by a larger theme: thoughtful tools for modern life.

### 6. Keep the site calm

Avoid visual clutter. Let spacing, type, and strong imagery carry the experience.

## Typography Direction

The typography should balance editorial personality with product clarity.

Suggested direction:

- Use a refined serif for large editorial moments.
- Use a clean sans-serif for UI, navigation, captions, and body text.
- Keep body copy highly readable.
- Avoid using too many font families.

Potential pairings:

- Cormorant Garamond or a similar editorial serif for display moments
- Inter, Neue Haas Grotesk, Red Hat Display, or a similar sans-serif for interface and body

Do not overuse the serif. It should create moments of elegance, not reduce usability.

## Color Direction

The color system should support:

- Soft pinks
- Lavender
- Warm cream / off-white
- Muted charcoal
- Gentle orange accents
- Subtle gradients

Use orange carefully as an accent, not as the main brand color.

The palette should feel warm, editorial, and modern — not candy-like or childish.

## Motion Direction

Motion should be subtle and intentional.

Good uses of motion:

- Hero introduction
- Showing "multiple hats" or different roles
- Smooth transitions between project categories
- Hover states
- Light reveal animations
- Soft gradient movement
- Scroll-based editorial pacing

Avoid:

- Constant looping distractions
- Overly complex animation systems
- Gimmicky interactions
- Motion that makes the site harder to navigate

If implementing motion, prefer tasteful, lightweight animation with strong performance.

## Accessibility Requirements

All work should respect accessibility basics:

- Text must have sufficient contrast.
- Interactive elements need visible focus states.
- Images need meaningful alt text where appropriate.
- Motion should not be excessive.
- Layout should work well on mobile.
- Navigation should be keyboard accessible.
- Buttons and links should have clear labels.

Do not sacrifice readability for aesthetics.

## Responsive Behavior

The portfolio must work beautifully on:

- Desktop
- Laptop
- Tablet
- Mobile

Desktop can feel more editorial and spacious.

Mobile should be simple, clear, and fast. Avoid fragile layouts or tiny text.

Case studies should remain easy to scan on small screens.

## Design Token Guidance

Use tokens for:

- Color
- Typography
- Spacing
- Radius
- Border
- Shadow
- Motion duration
- Breakpoints

Separate primitive tokens from semantic tokens when possible.

Primitive examples:

- `color.pink.100`
- `color.lavender.200`
- `color.orange.500`
- `space.4`
- `radius.lg`

Semantic examples:

- `color.background.primary`
- `color.text.default`
- `color.accent.warm`
- `color.surface.soft`
- `color.border.subtle`

## Do Not Do

Do not:

- Make the portfolio look like a generic AI startup template.
- Overload the homepage with too much text.
- Use loud neon AI visuals.
- Add unnecessary decorative icons.
- Make every section a card.
- Hide important work behind clever interactions.
- Prioritize novelty over clarity.
- Rewrite Belle's voice into corporate jargon.

## Decision-Making Rule

When unsure, choose the option that feels:

1. Clearer
2. More editorial
3. More refined
4. Easier to scan
5. More aligned with AI/product design opportunities

Clarity comes first. Taste comes second. Novelty comes last.
